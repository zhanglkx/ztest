🗺️ 第3课：CSS Grid 网格区域

掌握grid-area和grid-template-areas的使用

🎯 grid-template-areas - 可视化布局

grid-template-areas 是Grid布局中最直观的功能，它允许你通过ASCII艺术的方式定义布局结构。

Header 头部
Sidebar 侧边栏
Main Content 主要内容
Footer 底部
/* CSS */
.layout-container {
    display: grid;
    grid-template-areas: 
        "header header header"    /* 第一行：头部占据3列 */
        "sidebar main main"       /* 第二行：侧边栏1列，主内容2列 */
        "footer footer footer";   /* 第三行：底部占据3列 */
    grid-template-columns: 200px 1fr 1fr;
    grid-template-rows: 80px 300px 60px;
    gap: 15px;
}

/* HTML中对应的元素 */
.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }
.footer { grid-area: footer; }
💡 语法规则
  • 每个字符串代表一行
  • 字符串中的每个名称代表一个网格单元
  • 相同名称的单元会合并成一个区域
  • 使用点号 (.) 表示空白单元

📐 网格线命名与定位

除了使用区域名称，还可以通过网格线命名来精确控制元素位置。

A (start/line3)
B (line3/end, row1/row2)
C (start/line2)
D (line2/line3)
/* 定义带命名的网格线 */
.grid-container {
    display: grid;
    grid-template-columns: [start] 1fr [line2] 1fr [line3] 1fr [end];
    grid-template-rows: [row1-start] 100px [row1-end row2-start] 100px [row2-end];
}

/* 使用命名网格线定位 */
.item-a {
    grid-column: start / line3;    /* 从start线到line3线 */
    grid-row: row1-start;          /* 第一行 */
}

.item-b {
    grid-column: line3 / end;      /* 从line3线到end线 */
    grid-row: row1-start / row2-end; /* 跨越两行 */
}
🔧 grid-column 和 grid-row 定位

使用数字或span关键字进行网格定位：

跨3列 (1/4)
跨2行 (1/3)
跨2列 (1/3)
单格 (3)
跨4列 (1/5)
/* 使用数字定位 */
.item-1 { 
    grid-column: 1 / 4;    /* 从第1列线到第4列线 */
    grid-row: 1;           /* 第1行 */
}

.item-2 { 
    grid-column: 4;        /* 第4列 */
    grid-row: 1 / 3;       /* 从第1行到第3行 */
}

/* 使用span关键字 */
.item-5 { 
    grid-column: span 4;   /* 跨越4列 */
    grid-row: 3;           /* 第3行 */
}
📍 数字定位
grid-column: 1 / 3;

从第1列线到第3列线（占2列）

🔄 span关键字
grid-column: span 2;

从当前位置跨越2列

↩️ 负数定位
grid-column: -3 / -1;

从倒数第3列线到最后一列线

🎯 简写属性
grid-area: 1 / 2 / 3 / 4;

行起始/列起始/行结束/列结束

🚀 高级技巧与最佳实践
📱 响应式区域

使用媒体查询改变grid-template-areas，实现响应式布局重排。

🎨 语义化命名

使用有意义的区域名称，如"header"、"main"，便于理解和维护。

⚡ 性能优化

grid-template-areas比复杂的定位计算性能更好。

🔍 调试友好

浏览器开发工具能清晰显示命名区域，便于调试。

🎯 实战建议

对于复杂布局，建议先用grid-template-areas快速搭建结构，再用grid-column/grid-row进行细节调整。这种组合使用的方式既直观又灵活。

/* 响应式布局示例 */
.layout {
    display: grid;
    grid-template-areas: 
        "header header header"
        "sidebar main aside"
        "footer footer footer";
    grid-template-columns: 200px 1fr 200px;
}

@media (max-width: 768px) {
    .layout {
        grid-template-areas: 
            "header"
            "main"
            "sidebar"
            "aside"
            "footer";
        grid-template-columns: 1fr;
    }
}
