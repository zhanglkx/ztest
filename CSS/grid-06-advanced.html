🚀 第6课：CSS Grid 高级特性

探索隐式网格、自动流、子网格等高级功能

🔄 隐式网格 (Implicit Grid)

当网格项目超出明确定义的网格轨道时，Grid会自动创建隐式轨道来容纳这些项目。

.grid-container {
    display: grid;
    grid-template-columns: repeat(3, 100px);  /* 定义3列 */
    grid-template-rows: 80px 80px;            /* 定义2行 */
    
    /* 控制隐式轨道的大小 */
    grid-auto-rows: 60px;        /* 隐式行高60px */
    grid-auto-columns: 120px;    /* 隐式列宽120px */
}
📏 grid-auto-rows
grid-auto-rows: 60px;

设置隐式创建的行轨道的高度

📐 grid-auto-columns
grid-auto-columns: 120px;

设置隐式创建的列轨道的宽度

➡️ 自动流 (Grid Auto Flow)

grid-auto-flow 控制自动放置的网格项目的流向。

grid-auto-flow: column 效果

.grid-container {
    display: grid;
    grid-template-columns: repeat(3, 100px);
    grid-template-rows: repeat(2, 80px);
    
    /* 改变自动流向 */
    grid-auto-flow: column;  /* 优先填充列 */
    grid-auto-columns: 100px; /* 新列的宽度 */
}
⬇️ row (默认)
grid-auto-flow: row;

按行填充，先填满一行再到下一行

➡️ column
grid-auto-flow: column;

按列填充，先填满一列再到下一列

🔄 dense
grid-auto-flow: row dense;

尽量填满网格空隙，可能改变项目顺序

🧩 密集排列 (Dense Packing)

使用dense关键字可以让Grid尽量填满所有空隙，特别适合有不规则大小项目的布局。

.dense-grid {
    display: grid;
    grid-template-columns: repeat(4, 80px);
    grid-auto-flow: row dense;  /* 密集排列 */
    gap: 10px;
}

.wide-item {
    grid-column: span 2;  /* 占据2列 */
}

.tall-item {
    grid-row: span 2;     /* 占据2行 */
}
⚠️ 注意事项

使用dense可能会改变项目的视觉顺序，这可能影响屏幕阅读器用户的体验。请谨慎使用。

🔗 子网格 (Subgrid) - 未来特性

subgrid允许网格项目继承其父网格的轨道定义，目前浏览器支持有限。

卡片标题 1

这是一些内容文本，可能长度不同。在子网格中，所有卡片的对应部分会对齐。

更长的卡片标题 2

内容可能更短。

标题 3

这里有更多的内容文本，展示了不同长度的内容在子网格中如何对齐。

/* 子网格语法 (未来特性) */
.parent-grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 20px;
}

.card {
    display: grid;
    grid-template-rows: subgrid;  /* 继承父网格的行轨道 */
    grid-row: span 3;             /* 跨越3行 */
}

/* 当前的替代方案 */
.card-current {
    display: grid;
    grid-template-rows: auto 1fr auto;  /* 手动定义行 */
    gap: 10px;
}
🔮 浏览器支持

子网格目前只有Firefox完全支持。在其他浏览器中，可以使用flexbox或手动定义网格轨道作为替代方案。

🛠️ 实用高级技巧
🎯 网格线命名模式

使用一致的命名模式，如 [header-start] [header-end]，提高代码可读性。

🔄 动态网格项目

结合JavaScript动态添加网格项目，Grid会自动调整布局，无需重新计算。

📱 容器查询准备

Grid与未来的容器查询结合使用，将提供更强大的响应式布局能力。

⚡ 性能优化

合理使用隐式网格和自动流，减少不必要的显式定义，提升性能。

/* 高级网格模式 */

/* 1. 内容驱动的网格 */
.content-grid {
    display: grid;
    grid-template-columns: 
        repeat(auto-fit, minmax(min-content, max-content));
    justify-content: center;
    gap: 20px;
}

/* 2. 智能卡片网格 */
.smart-cards {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
    grid-auto-rows: minmax(200px, auto);
    gap: 20px;
}

/* 3. 瀑布流效果（需要JavaScript配合） */
.masonry-grid {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
    grid-auto-rows: 10px; /* 小单位便于精确控制 */
    gap: 10px;
}

/* 4. 复杂响应式布局 */
.advanced-layout {
    display: grid;
    grid-template: 
        "header header header" auto
        "nav main aside" 1fr
        "footer footer footer" auto
        / 200px 1fr 150px;
    gap: 20px;
    min-height: 100vh;
}
