常见布局模式和实战案例

🏛️ 圣杯布局 (Holy Grail Layout)

经典的头-导航-主内容-侧边栏-尾五区域布局，Grid让它变得简单。

Header 头部
Navigation 导航
Main Content 主内容
Aside 侧边栏
Footer 底部
.holy-grail {
    display: grid;
    grid-template: 
        "header header header" 80px
        "nav main aside" 1fr
        "footer footer footer" 60px
        / 200px 1fr 150px;
    gap: 15px;
    min-height: 100vh;
}

.header { grid-area: header; }
.nav { grid-area: nav; }
.main { grid-area: main; }
.aside { grid-area: aside; }
.footer { grid-area: footer; }

/* 响应式调整 */
@media (max-width: 768px) {
    .holy-grail {
        grid-template: 
            "header" auto
            "nav" auto
            "main" 1fr
            "aside" auto
            "footer" auto
            / 1fr;
    }
}

.card-grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
    gap: 25px;
}

.card {
    background: white;
    border-radius: 15px;
    overflow: hidden;
    box-shadow: 0 8px 25px rgba(0, 0, 0, 0.1);
    transition: transform 0.3s ease;
}

.card:hover {
    transform: translateY(-5px);
}

🎯 实战技巧总结
💡 最佳实践
  • 移动优先：从单列布局开始，逐步增加复杂度
  • 语义化命名：使用有意义的区域名称，便于维护
  • 灵活单位组合：混合使用fr、px、minmax()等单位
  • 适度使用媒体查询：Grid本身就很响应式，减少断点数量
  • 考虑可访问性：确保键盘导航和屏幕阅读器友好
/* 通用响应式模板 */
.layout-container {
    display: grid;
    gap: 20px;
    padding: 20px;
    min-height: 100vh;
}

/* 桌面端 */
@media (min-width: 768px) {
    .layout-container {
        grid-template-columns: 250px 1fr 200px;
        grid-template-rows: auto 1fr auto;
        grid-template-areas: 
            "header header header"
            "sidebar main aside"
            "footer footer footer";
    }
}

/* 平板端 */
@media (max-width: 767px) and (min-width: 481px) {
    .layout-container {
        grid-template-columns: 1fr;
        grid-template-areas: 
            "header"
            "main"
            "sidebar"
            "aside"
            "footer";
    }
}

/* 手机端 */
@media (max-width: 480px) {
    .layout-container {
        padding: 10px;
        gap: 15px;
    }
}
